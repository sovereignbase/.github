# Code of Conduct

Sovereignbase is a technical project focused on building precise, interoperable, user-sovereign systems.

Discussion is expected to be rigorous, direct, and technically grounded.

## Principles

- Debate ideas, not people.
- Critique designs, not identity.
- Prefer evidence and reasoning over authority.
- Represent others’ positions accurately before criticizing them.
- Keep discussion focused on the technology, architecture, and specifications.

Strong disagreement is welcome.  
Personal hostility is not.

## Technical Standards

- Claims should be backed by reasoning, references, or implementation evidence.
- Discussions should clearly distinguish between:
  - specification-level requirements
  - implementation details
  - operational tradeoffs
  - personal preferences
- Security, privacy, and decentralization claims must not be exaggerated.
- Criticism should aim to improve correctness, clarity, or interoperability.

## Unacceptable Behavior

- Harassment, intimidation, or discriminatory language
- Personal attacks, insults, or sustained hostility
- Misrepresenting another person’s argument
- Intentional misinformation
- Repeated bad-faith argumentation
- Publishing private or sensitive information without consent
- Disruptive behavior that consistently prevents productive technical discussion

## Enforcement

Maintainers may remove comments, close discussions, reject contributions, or restrict participation if behavior violates this policy.

Sovereignbase is a project for building serious systems.
If discussion repeatedly undermines that goal, it will be moderated.

## Scope

This Code of Conduct applies to:
- Issues
- Pull requests
- Discussions
- Documentation discussions
- Community spaces and other official Sovereignbase communication channels
